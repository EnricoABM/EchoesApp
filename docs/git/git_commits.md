# Padrão de Commit utitlizado durante o desenvolvimento

---

* feat: Uma nova funcionalidade (ex: criação da tela de login).

* fix: Correção de um erro/bug.

* docs: Alterações apenas na documentação (README, comentários).

* style: Mudanças de formatação que não afetam a lógica (espaços, ponto e vírgula).

* refactor: Mudança no código que não corrige erro nem adiciona funcionalidade (melhoria de performance ou limpeza).

* test: Adição ou correção de testes.

* chore: Atualizações de tarefas de build, pacotes ou ferramentas auxiliares (ex: instalar uma biblioteca).
